# S3 Bucket Configuration for AWS Cloud Monitoring

## Overview

This setup configures an Amazon S3 bucket to securely store AWS CloudTrail logs for centralized monitoring and ingestion into Splunk SIEM.

The S3 bucket acts as the primary storage layer in the AWS SOC monitoring pipeline.

---

# Objective

The purpose of this setup is to:

- Store AWS CloudTrail logs
- Enable centralized AWS activity monitoring
- Support Splunk log ingestion through SQS notifications
- Maintain long-term audit visibility for security investigations

---

# Architecture Flow

AWS CloudTrail → S3 Bucket → SQS Queue → Splunk Enterprise

---

# Step 1 — Open Amazon S3 Service

1. Login to the AWS Console
2. Search for `S3`
3. Open the Amazon S3 service
4. Click `Create bucket`

# Step 2 — Configure Bucket Settings

Configure the following settings:

| Setting | Value |
|---|---|
| AWS Region | US East (Ohio) |
| Bucket Type | General purpose |
| Bucket Name | nextwork-s3-bucket-soc |

The bucket name must be globally unique.

---

  <img src="../../assets/aws-cloud-monitoring/s3-setup/create-bucket-page.png" width="900">

---

# Step 3 — Configure Object Ownership

Select:

- ACLs disabled (recommended)

This enables bucket owner enforced access control.

---

<p align="center">
  <img src="../../assets/aws-cloud-monitoring/s3-setup/object-ownership.png" width="900">
</p>

---

# Step 4 — Configure Public Access Settings

Keep all public access blocked.

Recommended settings:

- Block all public access = Enabled

This prevents unintended public exposure of CloudTrail logs.

---

<p align="center">
  <img src="../../assets/aws-cloud-monitoring/s3-setup/object-ownership.png" width="900">
</p>

---

# Step 5 — Configure Bucket Versioning

Bucket versioning was configured as:

- Disabled

Versioning can be enabled later for advanced recovery and retention use cases.

# Step 6 — Configure Default Encryption

Server-side encryption was enabled using:

- SSE-S3 (Amazon S3 managed keys)

The following security controls were implemented:

- Public access blocked
- Server-side encryption enabled
- Centralized log storage
- Dedicated monitoring bucket
