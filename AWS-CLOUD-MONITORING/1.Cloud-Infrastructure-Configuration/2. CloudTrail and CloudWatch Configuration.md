# CloudTrail and CloudWatch Configuration for AWS Monitoring

## Overview

This setup configures AWS CloudTrail and CloudWatch Logs for centralized AWS activity monitoring and security event visibility.

CloudTrail captures AWS API activities while CloudWatch stores and streams logs for monitoring and integration with Splunk SIEM.

---

# Objective

The purpose of this configuration is to:

- Capture AWS management events
- Monitor IAM and AWS account activities
- Store logs securely in S3
- Stream logs into CloudWatch
- Support Splunk SIEM ingestion
- Enable security investigation and auditing

---

# Architecture Flow

AWS Services → CloudTrail → S3 Bucket → CloudWatch Logs → SQS Queue → Splunk Enterprise

---

# Step 1 — Open CloudTrail Service

1. Login to AWS Console
2. Search for `CloudTrail`
3. Open the CloudTrail service
4. Click `Create trail`

# Step 2 — Configure Trail Attributes

Configure the following settings:

| Setting | Value |
|---|---|
| Trail Name | nextwork-trail |
| Storage Location | Existing S3 bucket |
| S3 Bucket | nextwork-s3-bucket-soc |
| Multi-region Trail | Enabled |

This configuration allows CloudTrail to capture activities from multiple AWS regions.

---

  <img src="/assets/cloudtrail-trail-attributes.png" width="900">

# Step 3 — Enable CloudWatch Logs

CloudWatch integration was enabled to monitor AWS events in real time.

Configure:

| Setting | Value |
|---|---|
| CloudWatch Logs | Enabled |
| Log Group | aws-cloudtrail-logs-nextwork |
| IAM Role | Auto-created by AWS |

This allows CloudTrail logs to be streamed into CloudWatch Logs.

---

# Step 4 — Configure Log Events

Management events were enabled for monitoring AWS account activities.

Configured options:

- Management Events = Enabled
- Read Events = Enabled
- Write Events = Enabled

This captures activities such as:

- IAM policy changes
- User creation
- EC2 creation
- Security group modifications
- Root account activities

# Step 6 — Verify Trail Creation

After creating the trail:

1. Open the `Trails` section
2. Verify trail status
3. Confirm logging is enabled
4. Check S3 bucket integration
5. Verify CloudWatch Log Group

Trail status should display as:

- Logging = Enabled

# Step 7 — Generate AWS Events

To test monitoring:

1. Open AWS Secrets Manager
2. Modify or access a secret
3. Save the changes

This generates AWS API events that CloudTrail records.
