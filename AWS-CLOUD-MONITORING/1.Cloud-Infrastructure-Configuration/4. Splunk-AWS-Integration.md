
# Splunk AWS Integration for Cloud Monitoring

## Overview

This setup integrates AWS CloudTrail logs into Splunk Enterprise using:

- AWS CloudTrail
- Amazon S3
- Amazon SQS
- Splunk Add-on for AWS

The integration enables centralized monitoring, alerting, and investigation of AWS activities inside the SOC environment.

---

# Objective

The purpose of this setup is to:

- Collect AWS CloudTrail logs
- Forward AWS logs into Splunk
- Monitor IAM and cloud activities
- Validate end-to-end AWS event ingestion
- Build cloud detection and alerting capabilities

---

# Architecture Flow

AWS Action → CloudTrail → S3 Bucket → SQS Queue → Splunk Enterprise

---

# AWS Services Used

| Service | Purpose |
|---|---|
| AWS CloudTrail | Capture AWS API activity |
| Amazon S3 | Store CloudTrail logs |
| Amazon SQS | Notify Splunk about new logs |

# Step 8 — Configure SQS Access Policy

If S3 cannot send notifications to SQS:

Navigate to:

AWS Console → SQS → Queue → Access Policy

Add permissions allowing:

- S3 to send messages
- Queue owner full access

Example configuration:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "OwnerFullAccess",
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::ACCOUNT_ID:root"
      },
      "Action": "SQS:*",
      "Resource": "QUEUE_ARN"
    },
    {
      "Sid": "AllowS3SendMessage",
      "Effect": "Allow",
      "Principal": {
        "Service": "s3.amazonaws.com"
      },
      "Action": "SQS:SendMessage",
      "Resource": "QUEUE_ARN",
      "Condition": {
        "ArnLike": {
          "aws:SourceArn": "S3_BUCKET_ARN"
        }
      }
    }
  ]
}
```

Save the policy.

# Event Verification Workflow

## Step 1 — Verify CloudTrail Captured Event

Navigate to:

CloudTrail → Event History

Search:

```text
AttachUserPolicy
```

If found:

✅ CloudTrail recorded the activity

---

# Step 2 — Verify Log File Reached S3

Navigate to:

S3 → CloudTrail Bucket → AWSLogs

Example path:

```text
AWSLogs/<account-id>/CloudTrail/us-east-1/YYYY/MM/DD/
```

Example:

```text
AWSLogs/922806890303/CloudTrail/us-east-1/2026/05/14/
```

You should see:

```text
.json.gz
```

CloudTrail log files.

---

# Step 3 — Verify Event Exists in Log File

Download a:

```text
.json.gz
```

file.

Extract it.

Open JSON.

Search for:

```text
AttachUserPolicy
```

If found:

✅ CloudTrail successfully delivered logs into S3

---

# Step 4 — Verify SQS Received Notification

Navigate to:

SQS → Queue → Send and Receive Messages

Click:

Poll for messages

If messages appear:

✅ S3 notification successfully reached SQS

---

# Step 5 — Verify Splunk Consumed Queue

If Splunk is working correctly:

- SQS messages disappear
- Queue message count decreases
- Messages received increases
- Messages deleted increases

Navigate to:

SQS → Monitoring

This confirms Splunk successfully consumed the queue.
